Proceso de pruebas

Para el proceso de pruebas se designaron ciertas tareas a cada integrante del equipo.
Las pruebas que se realizaron son las siguientes:

Objetivo de la Prueba Verificar que el inicio de sesión en la aplicación web funcione correctamente, permitiendo solo a usuarios con correo institucional, aplicando validación de contraseña y sin mostrar errores.
Responsable Josue Erick Coyotl Rojas
Pasos 1. Ambiente de la prueba:
  • Entorno de pruebas: Servidor de desarrollo.
  • Navegador utilizado: Google Chrome versión 95.0.4638.54.
  • Usuario de prueba: Usuario de correo institucional válido.
  • Base de datos de prueba con datos de prueba disponibles.
2. Particionar el sistema en módulos de unidades lógicas fáciles de probar:
  • Módulo de inicio de sesión.
  • Módulo de gestión de usuarios.
  • Módulo de validación de contraseñas.
3. Aspectos a considerar:
  • Rutinas de excepción:
    Se verificó que el sistema maneja excepciones de manera adecuada en caso de fallos inesperados.
  • Rutinas de error:
    Se intentó el inicio de sesión con credenciales inválidas para asegurarse de que se maneje correctamente el error de autenticación.
  • Manejo de parámetros:
    Se verificó que los parámetros de inicio de sesión se envían y reciben correctamente.
  • Validaciones:
    Se introdujeron datos de entrada válidos y no válidos para verificar que se aplican las validaciones adecuadas. Se confirmó que el sistema impide el inicio de sesión con correos no institucionales.
  • Valores válidos:
    Se inició sesión con credenciales válidas para asegurarse de que el proceso funcione sin problemas.
  • Valores límites:
    Se probaron contraseñas con la longitud mínima y máxima permitida para verificar que no haya problemas en los extremos.
  • Mensajes posibles:
    Se verificó que los mensajes de inicio de sesión, errores y validación sean coherentes y fáciles de entender. Se confirmó que los mensajes de error proporcionan información útil para la resolución de problemas.
Técnica El inicio de sesión en la aplicación web ha sido probado y cumple con los requisitos establecidos, excepto por la validación de caracteres máximos y mínimos en la contraseña.
Objetivo de la Prueba Verificar que la función de realizar publicaciones en el foro funcione correctamente, mostrando la publicación en el feed con el nombre de usuario, fecha/hora y permitiendo la visualización de imágenes adjuntas.
Responsable Edward Sánchez Gil
Pasos 1. Ambiente de la Prueba:
  • Navegador a utilizar
    Usuario de prueba: Usuario registrado válido.
    Se ha preparado un conjunto de datos de prueba que incluye usuarios y publicaciones ficticias.
2. Particionar el sistema en módulos de unidades lógicas fáciles de probar:
  • Módulo de publicación en el foro.
    Módulo de visualización del feed.
    Módulo de carga de imágenes.
3. Aspectos a Considerar:
  • Creación de una publicación:
    Se creó una nueva publicación en el foro utilizando un usuario de prueba.
    Se verificó que se pueden incluir texto e imágenes.
  • Visualización de la publicación:
    Se confirmó que la publicación creada se muestra correctamente en el feed.
    Se verificó que el nombre de usuario que publico corresponda al de la publicación.
    Se comprobó que la fecha y hora de la publicación sean correctas y actualizadas automáticamente.
  • Visualización de imágenes:
    Se adjuntó una imagen a la publicación y se verificó que se muestra correctamente en el feed.
Técnica
  • La función de realizar publicaciones en el foro funciona según lo esperado al igual que se muestran en el feed con el nombre del usuario y la fecha/hora correctos, también las imágenes adjuntas se visualizan correctamente en las publicaciones.
  • En ocasiones al realizar una publicación se muestra como si la publicación hubiera sido realizada hace 23 horas, pero esta se corrige en cuanto se realiza una nueva publicación.
