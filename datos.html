Datos locales, remotos y offline.

Durante el desarrollo de esta actividad, se tenía planeado usar el proyecto “castor senpai” para el mismo, internamente hubo conflictos que impidieron el uso del mismo, por temas de compatibilidad entre flask y el service worker, no se pudo llevar a cabo lo planeado, impidiendo la adecuada implementación del service workwer en dicha aplicación.

Datos remotos:

En la aplicación castor senpai se implementan datos remotos porque están almacenados en una base de datos Mysql en el servidor, para acceder a ellos se tiene que hacer una solicitud al servidor controlado por flask y desarrollado en Python para que este haga las consultas y procese los datos para regresarlos en un formato JSON al cliente y de esta forma poder mostrarlos al usuario.

Datos locales:

Dentro de la app desarrollada, contiene un archivo de manifiesto (manifest.js), dicho archivo tiene la función de cargar la configuración inicial de la página, y algunos archivos de la misma como el icono de la aplicación, esta configuración permite instalar la aplicación web como aplicación del dispositivo, gracias a ello se puede acceder a la aplicación sin necesidad de que previamente se haya tenido acceso a la misma desde el navegador, una vez echo esto, son considerados archivos locales en el dispositivo, ya que se encuentran almacenados internamente en el mismo.

Datos offline:

Con la ayuda del service worker, durante el primer inicio de la aplicación los archivos que en un principio se encuentran en el servidor web se cargan en la cahe, para su uso posterior, en caso de que la app se vuelva a abrir el service worker interceptara los datos para verificar si están cargados en la cache, en caso de ser cierta la condición anterior, serán cargados desde ahí. Estos datos al estar cargados en la cache son considerados como datos offline, ya que se obtienen en la cache y no del servidor.
